14 // This is an involved integration test of
15 // haplotype and genotype matrices, and it takes
16 // some time to run.
17 // We simulate a population of N=1000 diploids to equilibrium.
18 // We then construct a haplotype and genotype matrix from a large
19 // sample of diploids. We tests that row and sum columns from the two
20 // matrix types agree. Once those agreement tests pass, we compare the
21 // haplotype matrix to the row/sum columns of a sample based on
22 // KTfwd::sample_separate (fwdpp/sugar/sampling.hpp) for the same diploids.
23 // These last checks ensure that two independent pieces of code, written
24 // at different times, give the same results.
25 //
26 // Then, to really make sure, we do the above tests 1,000 times, evolving
27 // our population 100 generations in between each test.

218 // the default behavior is that GSL raises

222 // and then turn error handler back on.

225  spoptype pop(1000); // a monomorphic population

231  auto m = haplotype_matrix(pop, indlist, keys.first, keys.second);
232  BOOST_REQUIRE(keys.first.empty());
